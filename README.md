# 🔍 Document-search-engine

Асинхронный веб-сервис для полнотекстового поиска по текстам документов. Проект включает в себя полный комплекс современных инженерных решений: сквозную контейнеризацию всех компонентов, развитый асинхронный стек технологий и законченный пакет функциональных тестов.

---

## 🛠️ Технологический стек

- **Backend:** Python 3.11, FastAPI (микрофреймворк)
- **База данных:** PostgreSQL 15 + асинхронный драйвер `asyncpg` + SQLAlchemy 2.0 ORM
- **Поисковый индекс:** Elasticsearch 7.17 (с поддержкой русской морфологии и падежей)
- **Контейнеризация:** Docker & Docker Compose
- **Тестирование:** Pytest + HTTPX (AsyncClient)

---

## 🚀 Как запустить проект (Пошагово)

### Шаг 1: Клонирование репозитория
Склонируйте проект на свой компьютер и перейдите в корневую директорию:
```bash
git clone https://github.com/Oksana1992-hash/document-search-engine.git
cd document-search-engine
```

### Шаг 2: Настройка системных лимитов (для WSL / Linux)
Elasticsearch требует повышенных лимитов виртуальной памяти. Перед запуском контейнеров выполните команду в терминале вашей системы:
```bash
sudo sysctl -w vm.max_map_count=262144
```

### Шаг 3: Запуск инфраструктуры в Docker
Соберите образы и запустите все три контейнера (`db`, `elasticsearch`, `web`) в фоновом режиме:
```bash
docker-compose up -d --build
```
*Приложение автоматически дождется готовности баз данных. На старте FastAPI самостоятельно создаст таблицы в PostgreSQL и инициализирует индекс `documents` в Elasticsearch.*

### Шаг 4: Первичная загрузка и индексация данных (CSV)
В корне репозитория находится оригинальный файл `posts.csv` с массивом документов для импорта. 

Чтобы автоматически наполнить базу данных PostgreSQL и проиндексировать тексты в Elasticsearch, запустите скрипт импорта **внутри Docker-контейнера**:
```bash
docker-compose exec web python import_csv.py
```
*Скрипт автоматически очистит таблицы при повторном запуске, преобразует строки рубрик в нативные массивы PostgreSQL и отправит тексты в индекс Elasticsearch.*

## 📖 Документация API

После успешного выполнения `docker-compose up`, сервис доступен по адресу [http://localhost:8000](http://localhost:8000).

- **Интерактивная документация (Swagger UI):** [http://localhost:8000/docs](http://localhost:8000/docs) — *Рекомендуемый способ проверки интерактивных запросов*.
- **Спецификация OpenAPI (JSON):** Сохранена в корневом файле `docs.json` проекта. Вы можете открыть этот файл напрямую или импортировать в Postman/Insomnia.
- **Альтернативная документация (ReDoc):** [http://localhost:8000/redoc](http://localhost:8000/redoc) *(Зависит от доступности внешних CDN со скриптами отображения)*.

### Доступные эндпоинты:
1. `GET /search?query=текст` — Полнотекстовый поиск по индексу. Возвращает до 20 документов со всеми полями БД, упорядоченных по дате создания (`created_date` desc).
2. `DELETE /document/{doc_id}` — Удаление документа из PostgreSQL и поискового индекса Elasticsearch по его ID.

## 🧪 Запуск функциональных тестов

Тесты написаны с использованием асинхронного клиента и изолированы от глобального Event Loop. Для их безопасного запуска внутри Docker-контейнера выполните команду:

```bash
docker-compose exec web pytest tests/
```

## 🧹 Проверка качества кода

Код проекта полностью проверен на соответствие стандарту **PEP 8** и не содержит неиспользуемых импортов или синтаксических ошибок. Убедиться в чистоте кода можно с помощью линтера:

```bash
docker-compose exec web flake8 app/ tests/
```
